# SilentHook — Deception Engine

A bait-based deception engine that deploys decoy files, tokens and integrity monitors across your environment — silently alerting you the moment an insider or intruder takes the bait.

---

## Installation

Clone the repository:
```bash
git clone https://github.com/BENARD256/SilentHook.git
cd SilentHook
```

### Option 1 — Virtual Environment (Recommended)
```bash
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

### Option 2 — System-wide install
```bash
pip install -r requirements.txt --break-system-packages
```

> Option 2 is only recommended if you are not using a virtual environment and are on a system that restricts pip installs (e.g. Ubuntu 23+).

---

## Database Setup

The database schemas are located in the `db/` folder.

1. Create the database and user:
```sql
CREATE DATABASE dbbd;
CREATE USER 'your_db_user'@'localhost' IDENTIFIED BY 'your_password';
GRANT ALL PRIVILEGES ON dbbd.* TO 'your_db_user'@'localhost';
FLUSH PRIVILEGES;
```

> `your_db_user` and `your_password` can be anything you choose.  Just make sure they match what you set in `.env`.

2. Import the schema:
```bash
mysql -u your_db_user -p dbbd < db/silenthook_dump.sql
```

## Environment Variables

Create a `.env` file in the root of the project and fill in the following:

```env
# Security
JWT_SECRET_KEY="your_secret_key"

# Database
DB_USERNAME=your_db_user
DB_PASSWORD=your_password
DB_NAME=dbbd
DB_HOST=127.0.0.1

# Email
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USER=your_gmail@gmail.com
SMTP_PASS=your_app_password
```

### JWT Secret Key
Generate a strong random secret key using:
```bash
python3 -c "import secrets; print(secrets.token_hex(32))"
```
Copy the output and set it as `JWT_SECRET_KEY`.

### Email Setup (Gmail) — Required
`SMTP_USER` is the Gmail address that will send alert emails.
`SMTP_PASS` is **not your Gmail password** .It is a Google App Password.

App Passwords are 16-character codes generated by Google to allow apps to send emails on your behalf without using your actual password.

**To generate one:**
1. Enable 2-Step Verification on your Google account
2. Visit: [https://myaccount.google.com/apppasswords](https://myaccount.google.com/apppasswords)
3. Generate a password for **Mail**
4. Copy and paste it as `SMTP_PASS`

> Both `SMTP_USER` and `SMTP_PASS` are mandatory. Alerts will not be delivered without them.

## Port Configuration

### Flask Port
By default the app runs on port `5000`. To change it, update `apps.py`:
```python
app.run(debug=True, host='0.0.0.0', port=5000)
```

### MySQL Decoy Listener Port
SilentHook runs a fake MySQL server to catch attackers loading SQL dump baits. The default port is `3308` configurable in `config.py`:
```python
MYSQL_PORT = 3308
```

> Ensure port `3308` (or your chosen port) is open on your firewall:
> ```bash
> sudo ufw allow 3308/tcp
> ```
> Port `3306` is intentionally avoided since it is the real MySQL port. Using a different port prevents conflicts with your actual database server.

## Running the App

### Development
```bash
python3 apps.py
```
The app runs on `http://0.0.0.0:5000` by default, accessible on your local network.

### Production
In `apps.py` switch the config before running:
```python
app.config.from_object(ProductionConfig)
```
Then set `debug=False` and consider using a production WSGI server like Gunicorn:
```bash
gunicorn -w 4 -b 0.0.0.0:5000 apps:app
```
